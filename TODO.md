## CRITICAL — Breaks Core Promise

### 1. Light / Dark Mode System is Not Implemented
**Severity: CRITICAL (AGENTS.md: "This is a core product feature, not optional")**

- `constants/colors.ts` only exports a single dark `colors` object — no `lightColors` / `darkColors`
- `hooks/useTheme.ts` does not exist
- `store/useThemeStore.ts` does not exist
- Every screen and component imports `{ colors }` directly from constants, bypassing the theme system
- `app/_layout.tsx` hardcodes `StatusBar style="light"` with no theme adaptation
- Settings screen has no theme toggle wired up

**Files to create:**
- `hooks/useTheme.ts`
- `store/useThemeStore.ts`

**Files to update:**
- `constants/colors.ts` — add `lightColors` and `darkColors` exports, keep `colors` as dark alias
- `app/_layout.tsx` — adapt StatusBar to current theme
- Every component/screen — replace `import { colors }` with `const colors = useTheme()`

---

### 2. Health Entries Are Not Persisted to SQLite
**Severity: CRITICAL**

- `lib/db.ts` does not exist
- Health entries are stored only in Zustand memory (lost on every app restart)
- AGENTS.md requires SQLite for health entries and patterns

**Files to create:**
- `lib/db.ts` — SQLite helpers (init, insert, query, delete)

**Tables needed (from AGENTS.md):**
```sql
health_entries (id, timestamp, transcript, analysis, tags, severity, embedding)
health_patterns (id, entry_id, pattern_name, severity, description, recommendation, created_at)
family_members (id, name, relationship, public_key, connection_status, last_synced)
```

**Files to update:**
- `store/useHealthStore.ts` — wire addEntry() / removeEntry() through db.ts
- `store/useFamilyStore.ts` — wire addMember() through db.ts
- `app/analysis/result.tsx` — save button must persist to SQLite

### 3. QVAC Models Never Loaded at Startup

**Files to update:**
